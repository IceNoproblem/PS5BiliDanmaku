# PS5直播弹幕转发系统

PS5 原生直播系统，通过 DNS 重定向技术实现弹幕转发、RTMP推流等功能的完整解决方案。

## 核心功能

- 🎮 **PS5直播直连** - PS5原生直播功能支持
- 💬 **弹幕转发** - B站直播间弹幕实时转发到PS5
- 🎥 **RTMP推流** - 支持自定义RTMP服务器推流
- 🔒 **扫码登录** - B站扫码登录，安全稳定

## 🌐 DNS重定向的意义

### 为什么需要DNS重定向？

PS5直播系统之所以能工作，**核心技术是DNS重定向**。这里解释为什么它如此重要：

#### 1. **突破官方限制**
- PS5直播只能推送到官方支持的平台（如YouTube、Twitch）
- 通过DNS重定向，将PS5的直播请求劫持到本地服务器
- 无需修改PS5系统文件，完全通过网络层面实现

#### 2. **实现弹幕互动**
- PS5收不到B站弹幕的原因：两个系统数据通道完全隔离
- DNS重定向让我们可以：
  - 截获PS5的直播流
  - 获取B站弹幕数据
  - 通过扩展协议注入到PS5直播中

#### 3. **灵活的二次开发**
- 标准的RTMP推流协议
- 支持任何兼容RTMP的服务器
- 可以同时推流到多个平台

### DNS重定向工作原理

```
PS5直播请求流程：
1. PS5: 向 DNS 查询 "live-api.playstation.com"
2. DNS重定向: 返回本地IP (127.0.0.1)
3. PS5: 连接到本地RTMP服务器
4. 本地服务器: 接收直播流并处理
5. 弹幕注入: 从B站获取弹幕，合成到游戏画面
```

1、Windows系统

hosts 位于C:\windows\system32\drivers\etc文件里面，需要用管理员权限才可以编辑。

2、基于linux系统的路由可以使用下面方法，如OpenWrt固件、小米路由等

大部分智能路由器是定制的linux系统，使用root账户登录SSH，修改/etc/hosts，重启生效。

小米路由器 可以使用 小米wifi app 工具箱里面有个自定义HOST

3、斐讯路由

功能设置->高级设置->HOST配置或高级设置->系统设置->系统参数->hosts配置，使用标准hosts格式

4、潘多拉固件

网络->DHCP/DNS->HOSTS和解析文件->自定义DNS解析，根据提示添加即可

5、华硕梅林固件

首先登陆路由器后台，在系统管理 > 系统设置中开启【JFFS】和【SSH】，然后重启路由器。

使用SSH登陆路由器，在【/jffs/configs】下建立【dnsmasq.conf.add】文件，文件内容【addn-hosts=/jffs/configs/hosts】

在/jffs/configs下创建自己的hosts文件，按照标准hosts规则填写需劫持的域名，然后执行service restart_dnsmasq重启或者重启路由器即可生效。

6、爱快固件
网络设置->DNS设置->DNS设置->增加->填写要劫持的域名和IP->保存

7、极路由

在应用中心，安装自定义hosts，添加劫持，重启路由器即可

8、tomato固件

 在高级设置->DHCP/DNS>Dnsmasq自定义设置中，添加如下规则即可【address=/域名/IP】，截获DNS端口(UDP 53)打勾。

9、老毛子固件

内部网络->DHCP服务器->自定义HOSTS文件->添加一行->保存，重启生效。 作者：阿西西的日常 https://www.bilibili.com/read/cv22105285/?spm_id_from=333.1391.0.0&opus_fallback=1 出处：bilibili


## 功能说明

### 启动的三个核心组件

1. **nginx-rtmp** (端口 1935)
   - RTMP视频服务器
   - 接收PS5的直播流

2. **danmaku_forward.py** (端口 5000)
   - Web管理界面
   - B站弹幕获取
   - 弹幕合成与注入

3. **monitor_rtmp.py** (后台)
   - 监控RTMP服务状态
   - 实时推流统计
   - 性能监控

## 故障排查

### 启动失败

**问题**：启动时显示"找不到nginx.exe"
- **解决**：确保`_internal/nginx-rtmp/nginx.exe`文件存在，路径没中文。

**问题**：端口被占用
- 查看占用情况：`netstat -ano | findstr :5000`
- 结束进程：`taskkill /PID 端口号 /F`

**问题**：PS5无法连接、PS5开播后RTMP数据无变化，不显示推流码。
- 确认DNS劫持是否生效
- 在命令行运行 `nslookup contribute.live-video.net`
- 应该返回 本地IP

### 日志查看

启动程序会在命令行显示实时日志：
```
[主程序] Flask应用启动
[主程序] 监听 http://0.0.0.0:5000
[RTMP监控] 等待直播流...
```

## 常见问题

**Q: 这是安全的吗？**
A: 完全安全。我们只通过DNS重定向截获直播流，没有修改PS5系统文件，也不访问个人数据。

**Q: 是否支持直播录像？**
A: 支持。使用直播姬或者OSB拉去RTMP视频流录制。

**Q: 可以推流到其他平台吗？**
A: 可以。但是弹幕只能采集B站。

**Q: 需要破解PS5吗？**
A: 完全不需要。纯网络层面的DNS重定向，无需任何系统修改。
